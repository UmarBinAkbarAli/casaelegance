This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Static HTML/CSS/JavaScript multi-page website for "Casa Elegance" interior design services (Dubai). No runtime framework, no package manager — pages are generated by a Node.js build script and deployed as static files.

## Development

```bash
# Serve locally
python -m http.server 8080
# or
npx serve .

# Regenerate all HTML pages from templates
node build-pages.js
```

**Run `node build-pages.js` after every change to templates, partials, or data files.** The script reads `templates/` and overwrites the HTML pages in the project root in-place.

## Architecture

### Pages

Seven HTML files in the root — each is the build output, not a hand-edited source:

`index.html`, `about.html`, `services.html`, `projects.html`, `project-details.html`, `contact.html`, `cost-calculator.html`

Each page's `<main>` block is hand-authored in the HTML file; everything outside `<main>` (head, header, footer, partners strip) is injected by `build-pages.js`.

### Build system (`build-pages.js`)

- Reads shared partials from `templates/partials/` (header, footer, mobile-menu, search-modal, partners)
- Reads content data from `templates/data/` (projects.json, services.json, testimonials.json, site.json)
- Injects partials into `templates/layout.html` via `{{placeholder}}` tokens
- Renders page-specific dynamic sections (services grid, project cards, testimonials slider, contact form) by replacing `<section class="section-name">` placeholders inside each page's `<main>`
- Writes the result back to each HTML file

To add a new page: register it in the `pages` array in `build-pages.js`, author its `<main>` in the HTML file, then run the build.

### CSS (load order is fixed — do not reorder or merge)

| File | Contents |
|---|---|
| `vendor.css` | Third-party resets and Swiper overrides |
| `base.css` | CSS custom properties, typography, utilities |
| `components.css` | Reusable UI components (buttons, pills, cards) |
| `layout.css` | Page-level layout, grid, spacing |
| `pages.css` | Page-specific overrides |

Desktop breakpoint: `992px` (min-width). BEM naming: `mrittik-*`, `services-*`, `work-*`, `clone-*`.

### JavaScript (`script.js`)

Vanilla JS, no frameworks. Each feature is an independent `setup*()` function called unconditionally at load time. Uses `defer`. Key functions:

| Function | Responsibility |
|---|---|
| `setupMrittikHeader()` | Fixed nav, mobile menu toggle, search modal |
| `setupHeroSlider()` | Hero carousel (Swiper), autoplay, touch, dots |
| `setupScrollReveal()` | IntersectionObserver fade-in animations |
| `setupServices()` | Service card tilt/hover effects |
| `setupSlider()` | Testimonial slider with touch |
| `setupFaq()` | Accordion FAQ |

## Assets

- `assets/` — logos, icons, placeholder images (PNG/WebP)
- `assets/delivery/` — optimised content images (avif/webp/jpg triples) used by `build-pages.js` via `contentImageDeliveryMap`
- `assets/team/` — team member photos
- `assets/partners/` — partner SVG logos

## Conventions

- `data-*` attributes are the primary JS selector strategy — avoid `id`/`class` selectors that clash with existing BEM names.
- Touch handlers must use `{ passive: true }`.
- All photo images use `<picture>` with avif → webp → jpg sources. Logos, icons, SVGs, and the favicon are plain `<img>`.
- LCP image: `loading="eager"`, `fetchpriority="high"`, must be a local asset.
- `tmp-*` files in the project root are scratch references, not part of the site.
- `arsy-ref.css` is referenced in HTML but not present locally — intentional external reference, do not add it.

## Deployment

Pushes to `main` trigger a GitHub Actions workflow (`.github/workflows/deploy.yml`) that FTP-deploys the entire repo to `/public_html/` on the production server. Secrets: `FTP_SERVER`, `FTP_USERNAME`, `FTP_PASSWORD`.

## Mandatory validation after any UI change

Check on mobile (375px), tablet (768px), and desktop (1366px):

- Mobile menu opens/closes
- Hero section and slider
- Testimonial slider
- Our Team section
- Project cards and filter
- Contact form
- Footer
- Browser console — zero errors
